## 代码分析

分析代码文件，提取关键信息：

```
分析以下代码文件，提取：

文件: {{ FILE_PATH }}
语言: {{ LANGUAGE }}

请提取：
1. 模块职责（一句话描述）
2. 导出的函数（名称、签名、行号、描述）
3. 导出的类（名称、方法、属性、行号）
4. 类型定义（名称、定义、描述）
5. 内部依赖（项目内模块）
6. 外部依赖（第三方库）

输出 YAML 格式。
```

## 模块文档

生成模块说明文档：

```
根据以下信息生成模块文档：

模块名: {{ MODULE_NAME }}
模块路径: {{ MODULE_PATH }}
文件列表: {{ FILE_LIST }}
导出接口: {{ EXPORTS }}

生成内容包括：
1. 概述（2-3句）
2. 主要特性（列表）
3. 公开接口表格（包含源码链接）
4. 使用示例
5. 相关模块

格式：Markdown，使用中文
```

## 架构文档

生成项目架构概览：

```
根据以下项目结构生成架构文档：

项目名: {{ PROJECT_NAME }}
技术栈: {{ TECH_STACK }}
模块列表: {{ MODULES }}

生成内容：
1. Mermaid flowchart 架构图
2. 技术栈表格
3. 模块划分说明
4. 数据流 sequenceDiagram
5. 设计决策

Mermaid 要求：
- 使用 flowchart TB
- 节点 ID 只用字母数字
- 标签用双引号包裹
```

## API 文档

生成 API 参考文档：

```
为以下函数/类生成 API 文档：

{{ CODE_ITEMS }}

每个函数包含：
1. 标题行带源码链接: ### `name` [📄](file://path#L行号)
2. 简要描述
3. 函数签名代码块
4. 参数表格（参数名、类型、必填、说明）
5. 返回值
6. 使用示例

链接格式: [📄](file://{{ SOURCE_PATH }}#L{{ LINE_NUMBER }})
```
